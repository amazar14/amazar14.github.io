#配置库

#概要

此页面为管理软件配置用

#需求文档

#1. 说明

#1.1 基础配置库

  • 所有的 渠道 和 配置项 都存放于基础配置库
  • 基础配置库支持新增 配置项

#1.2 配置

  • 此配置为用户自定义 (即：基础配置库的子集。部分渠道 和 部分配置项 组成一个用户配置)
  • 用户可创建多个配置。

#1.3 渠道

  • 渠道必须从基础配置库中选择
  • 配置中应包含一个或多个渠道

#1.4 配置项

  • 配置项必须从基础配置库中选择
  • 一个配置项对应一个配置文件，该配置文件可替换和修改
  • 用户配置中的配置项会绑定渠道值，基础配置库中配置项不会绑定渠道值
  • 一个配置项应包含：
    • 配置名
    • 配置文件
    • 渠道编码（对应该配置中的渠道编码）
    • ...
配置基础数据示例
[
  {
    name: "配置一",
    channels: [
      { name: "渠道一", code: "渠道编码" },
      { name: "渠道二", code: "渠道编码" }
    ],
    configs: [
      { name: "配置项一", file: "xxx.xml", channelCode: "渠道编码" },
      { name: "配置项二", file: "xxx.xml", channelCode: "渠道编码" },
      ...
    ],
    ...
  },
  ...
]

#2. 流程

#2.1 配置

  1. 在基础配置库下创建配置，自定义配置名并提交
配置中的配置项和渠道

涉及到配置项的选择、是否继承、文件覆盖、重命名，新增数量等过多操作，所以不建议在新建配置中实现

#2.2 渠道

  • 以表格形式展示该配置所绑定渠道，可搜索
  • 新增渠道：渠道管理左上角新增渠道，弹框提供所有渠道（表格形式）自行添加
  • 删除渠道：表格操作列提供删除，删除前有提示

#2.3 配置项

  1. 配置项可通过渠道编码进行筛选，可搜索
  2. 配置 -> 配置项管理
    • 以卡片形式展示配置项相关信息
    • 新建
      • 左上角新增配置项，弹框选择基础库中配置项。（默认继承配置项名称和文件，可修改）
      • 可搜索渠道值的下拉框（此配置中的拥有的渠道值）
    • 查看
      • 卡片提供查看按钮，如果配置项信息过多可弹框显示所有信息
    • 修改
      • 卡片提供修改按钮，可复用新建的弹框
    • 删除
      • 卡片提供删除按钮，删除前有提示
  3. 基础配置库中可新建配置项
  • 文件上传、重命名
