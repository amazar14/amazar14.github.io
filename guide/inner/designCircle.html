#3. 设计软件通信

#3.1 左侧面板

#3.1.1 打开案例

使用案例
// send to homedesign
  {
    code: "DSCOMMUNITY_APPLY",
    params: {
      // 案例 hdf文件
      url: "https://cdn.xxxx.com/xxx.hdf",
      // 使用方式
      type: 1
    }
  }

type: 使用方式

值说明
1默认方式打开

#3.1.2 查看案例

查看案例
// send to homedesign
{
  code: "DSCOMMUNITY_DETAILS",
  params: {
    // todo：xml字符串OR指向xml的链接
    xml: "",
    // 封面图
    coverImgUrl: "",
    // 方案名
    schemeName: "",
    // 点击时的位置，软件需要用y值计算窗口位置
    position: {
      x: 0,
      y: 0
    }
  }
}

#3.2 投稿

#3.2.1 解析场景文件

  1. 点击投稿，需要弹出一个1000*800尺寸的窗口
打开投稿窗口
// send to homedesign
  {
    code: "COMM_OPENWINDOW_DESIGN_CIRCLE_CREATE",
    params: {
      url: ""
      type: "inner" | "outer"
      width: 1000,
      height: 800
    }
  }

type

type说明
inner嵌入窗口
outer外部浏览器
关闭窗口
// send to homedesign
{
  code: "COMM_CLOSEWINDOW_DESIGN_CIRCLE";
}

#3.2.2 确认投稿

确认投稿后

  1. 编辑的方案信息、空间信息后（只修改方案名、删减空间及空间下饰品），将 web 端将软件发送的场景信息以同样格式返回给软件，软件将json转为xml
  2. 设计软件将本地存储的场景文件及上一步中生成的场景xml上传
  3. 将文件在云端中的路径sceneFileUrl、 xmlFileUrl返回给 web
上传文件
// send to homedesign
  {
    code: "DSCOMMUNITY_UPLOAD_SCENE",
    param: {
      // 投稿场景时，设计软件返回的sceneFilePath
      __sceneFilePath__: 'C:\\Users\\1.hdf',
      // 其他同 DSCOMMUNITY_CREATE 中软件返回的场景信息
      fileId: "4DAB28DE-D3FE-4B2B-B26C-68D102A0BF85",
      fileName: "场景文档1",
      houseType: "",
      name: "场景文档1",
      resId: "",
      area: 123,
      ...
    }
  }

  // send to web 返回上传信息，***无论成功或失败，设计软件一定要返回这个信息***
  {
    code: "DSCOMMUNITY_UPLOAD_SCENE",
    status: true | false,
    message: '上传成功' | '上传失败',
    data: {
      sceneFileUrl: 'https://cdn.xxxx.com/xxx.hdf',
      xmlFileUrl: 'https://cdn.xxxx.com/xxx.xml'
    }
  }
