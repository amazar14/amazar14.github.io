#参数表导入

#概要

施工ing...

#地址

正式环境：http://help.jiajutech.com/tool/importParams?tokenId={tokenId}&cfgCode={cfgCode}&port={port}

测试环境：http://uat-help.jiajutech.com/tool/importParams?tokenId={tokenId}&cfgCode={cfgCode}&port={port}

参数说明
  1. tokenId: tokenId
  2. cfgCode: 配置code exp：ConstraintSetImport
  3. port: 用于和软件通信的websocket端口，需要软件传入

#参数配置

对于默认配置为json文件情况下，参数导入是场景通用的。

配置文件
{
  "ParamsSetImport": { 
    "title": "参数设置",
    "table": {
      "columns": [
        {
          "key": "baseSetting.name",
          "title": "名称"
        },
        {
          "key": "baseSetting.type",
          "title": "类型",
          "type": "Tag",
          "trans": {
            "1": {"label": "数值", "variant":"green"},
            "2": {"label": "文本", "variant":"gray"}
          }
        }
      ]
    },
    "websocket_code": "SYNC_DATA_PARAMS",
    "zyId_p": "81dacc733cdb45799b9d0f59a6a1827d",
    "zyId_d": "81dacc733cdb45799b9d0f59a6a1827d"
  },
  "ConstraintSetImport": {}
}

#说明

  • {ParamsSetImport}

    配置的code，应用会从url中cfgCode参数获取配置的code，然后根据这个code去配置文件中找到对应的配置。

  • title

    页面中的title，应用会根据这个字段显示在页面上。

  • table

    表格配置，应用会根据这个配置生成表格。

  • table.columns

    表格列配置，表格会根据这个配置显示字段

  • table.columns?.key

    数据中对应的字段，支持链式

  • table.columns.title

    表头

  • table.columns.type

    数据展示形式，目前支持Tag 和 文本，如果不想展示纯文本，可以配置这个字段，默认为文本

    1. Tag
    • trans 字段为必须，对应字段的值作为trans的key，label为tag的文本，variant为主题（red, orange, blue, purple, gray, green, ...）
    example{1,5}
    // 数据
{
  type: "1"
}
// 配置
{
  "key": "type",
  "title": "类型",
  "type": "Tag",
  "trans": {
    "1": {"label": "数值", "variant":"default"}
  }
}

    这样的配置会将数据中type字段的 1 值，显示在页面中为 数值 这样的标签

    1. 文本

    默认值，可不填

  • table.columns.key

    数据中对应的字段，支持链式

  • websocket_code

    websocket通信的code，应用会根据这个code去和设计软件通信。

  • zyId_p

    资源ID，应用会根据这个资源ID去后端获取对应的资源信息，也就是页面中左边的树数据。目前是直接写到这里，后面可能会根据cfgCode去后端获取

  • zyId_d

    同 zyId_p
