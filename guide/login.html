#3. 流程

合并
  1. 绑定手机号、修改手机号时，若手机号已绑定其他租户的账号，将触发合并。
  2. 一个手机号能够与多个不同租户的账号进行绑定。但是在同一个租户下，一个手机号只能绑定一个账号。

#3.1 微信扫码登录

  • 后端接口
    1. 获取临时ticket, ticket有效时间1分钟
    • /get/ticket
    1. 轮询接口，返回登录状态
    • /wxQrcodeStatus?ticket=173090909090
    1. 登录接口，根据前端传的code + ticket换取uniId查询用户登录
    • 没有这个用户就返回让页面显示注册，有就存到ticket，前端根据轮询状态用ticket获取
    • /wxlogin/ticket=173009090&uniId=987878789
    • 之前这个接口 /admin/account/getAccountByCode 可以根据code查找微信uniId并获取用户信息

#4. 软件登录

#4.1 缓存账号信息

登录成功后（记住密码），将账号列表全部缓存至软件

  • 如果账号已经登录过，第二次登录密码不一致时（1. 第一次未记住密码并且第二次记住密码。2. 忘记密码后修改密码）会导致修改密码的情况（与现有只支持存取的逻辑冲突），所以全部由web处理，软件只需要每次用web端传入的数据覆盖上一次缓存数据。
#example
  // send to HomdDesign
  {
    code: "CACHE_ACCOUNT",
    params: [{
      account: "account",
      password: "password"  // 加密处理
    }, ...]
  }
  // send to web
  {
    status: true,
    code: "CACHE_ACCOUNT",
    message: "error message",
    // 有用户信息就返回，没有返回null
    data: null                    
  }

#4.2 获取账号信息

第一次登录并记住密码缓存至软件，第二次登录需要从软件获取账号信息

#example
  // send to HomdDesign
  {
    code: "DISPLAY_ACCOUNTS",
    params: null
  }
  // send to web
  {
    status: true,
    code: "DISPLAY_ACCOUNTS",
    message: "error message",
    // 有用户信息就返回，没有返回空数组
    data: [{
      account: "account",
      password: "password" // 加密处理
    }, ...]
  },

#5. 统一登录

#5.1 地址

  • 正式环境：https://auth.ihomecloud.com.cn/auth/login?redir_url=aHR0cHMlM0ElMkYlMkZhZG1pbi5qaWFqdXRlY2guY29t&from=admin
  • 测试环境：https://uat-auth.ihomecloud.com.cn/auth/login?redir_url=aHR0cHMlM0ElMkYlMkZhZG1pbi5qaWFqdXRlY2guY29t&from=admin

参数说明

  1. redir_url：回调地址encode后base64加密
  2. from: 来源，用于区分是哪个系统登录
站点from
后台管理admin | pig
官网home | portal
videojiajuvideo
  • 其他值在后台管理 /admin/client/index 查询

#5.2 登录成功

登录成功后，会自动跳转到redir_url解码后的地址，并携带tokenId参数。

自动跳转回调
https://admin.ihomecloud.com.cn?tokenId=xxxxxx

子系统获取tokenId后，调用 GET：/auth/token/info?token=xxx 接口获取登录信息

#6. 作为iframe

#6.1 地址

  • 正式环境：https://auth.ihomecloud.com.cn/auth/login/inner?from=admin
  • 测试环境：https://uat-auth.ihomecloud.com.cn/auth/login/inner?from=admin

参数说明

  • from: 来源，用于区分是哪个系统登录，值同上。

#6.2 登录成功

登录成功后，会通过postMessage向父窗口发送消息，消息内容如下：

#example
{
  "type": "login",
  "tokenId": "xxxxx"
}

子系统获取tokenId后，调用 GET：/auth/token/info?token=xxx 接口获取登录信息
